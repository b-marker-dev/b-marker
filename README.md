## B-Marker Core Protocol

We believe that **"audio samples are the quanta of time."**

These pure functions form the foundational protocol for converting any time expression (seconds, milliseconds, musical beats) into a universal audio-sample language.

### ğŸ¯ Core Philosophy

### ğŸ§© Core Functions

This core protocol (`b-marker/core/`) is the cornerstone of the entire B-Marker ecosystem:

- `convert_markers`: converts marker positions between different audio sample rates.

- `time_to_samples`: converts time values to 44.1 kHz sample counts.

- `validate_jitter`: analyzes the time jitter of pulse sequences.

- `generate_simple_pulses`: generates binary pulse sequences from markers.

## B-Marker Project Description

### Project Overview

---

B-Marker is a profit-sharing management framework specifically designed for time and audio tool developers. It adopts a layered model combining open-source core functionality and commercial value-added services, aiming to balance the openness of open-source collaboration with the sustainability of commercial operations. The framework provides clear profit-sharing rules, standardized contribution evaluation mechanisms, and flexible licensing models, enabling developers to jointly build high-quality music tools while fairly sharing the commercial benefits generated by the project.

### Module Division

### Core Advantages

1. b-marker-tools (Commercial License): A toolset built on the core module, focusing on practical integration solutions for music tools (e.g., third-party platform integration, enterprise-level customization tools, batch processing plugins). It is free for individual developers and requires paid licensing for commercial use by enterprises.

2. b-marker-core (Open Source, MIT License): The foundational core module of the framework, providing core algorithms and basic functional components related to music tool development (e.g., beat detection, audio data processing, profit-sharing data modeling). It is fully open-source and free for all developers to use, modify, and integrate, laying a solid foundation for collaborative development.

3. b-marker-pro (Subscription License): The premium paid version of the framework, offering advanced value-added features such as cloud synchronization, professional audio analysis, exclusive sound effect libraries, and customized profit-sharing data dashboards. It adopts a tiered subscription model (individual/team/enterprise plans) to meet the diverse needs of professional developers and commercial teams.

### Target Users

Music tool developers, audio technology enthusiasts, independent developers focusing on music software, and enterprises requiring customized music tool solutions.

- Flexible Layered Licensing: The open-source core ensures the project's openness and scalability, while commercial modules guarantee the sustainability of project operations and contributors' income streams.

- Open & Transparent Profit-Sharing: All revenue sources, sharing ratios, and settlement cycles are fully disclosed. Profit-sharing is closely tied to contribution levels (including code development, testing, documentation, promotion, etc.), ensuring fair distribution of benefits.

- Standardized Collaboration Process: Equipped with comprehensive contributor agreements, PR templates, and issue handling mechanisms, reducing collaboration costs and improving development efficiency.

- Low Contribution Threshold: Supports both code and non-code contributions (e.g., documentation writing, community support, promotion). New contributors can participate in profit-sharing after completing one settlement cycle, with no lock-up period.

### License & Collaboration

We welcome developers worldwide to participate in the project's development. For contribution guidelines and detailed profit-sharing information, please refer to the CONTRIBUTING.md and profit-sharing/framework.md files in the repository.
